1. What kind of aggregation functions have you used in SQL or Python? What equivalent functions appear in DAX?

I have used aggregation functions like SUM, COUNT, AVERAGE/AVG, MIN, and MAX in SQL or Python. The DAX library has similar functions, such as SUM(), COUNT(), AVERAGE(), MIN(), and MAX(). These functions summarize data by adding values, counting rows, finding averages, or finding the smallest and largest values. Microsoft describes DAX aggregation functions as functions that calculate values like count, sum, average, minimum, or maximum.

2. Looking at the available text functions, what would you equate these to in SQL or Python?

DAX text functions are similar to SQL or Python string functions. For example, DAX has functions that can return part of a string, search for text, or combine text values. This is similar to using SQL functions like LEFT, RIGHT, CONCAT, and LEN, or Python string methods like slicing, .find(), and joining text.

3. What kinds of things can you do with information functions? How about logical functions?

Information functions help check what kind of value you are working with. For example, they can check whether a value is blank, text, a number, or an error. Logical functions help create decision-based formulas, like using IF, AND, or OR to return different results depending on whether a condition is true or false. Microsoft notes that information functions check whether values match expected types, while logical functions return information about values in an expression.
